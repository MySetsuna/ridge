# Ridge v0.0.5 — Agent Awareness Layer

> **版本**：0.0.5 | **目标**：从 Terminal Orchestration 升级到 Agent Awareness Layer
> **核心交付**：让系统知道 agent 是谁、在做什么、改了哪些文件

---

## 总览

### 架构变更

```text
Before (v0.0.2):    Pane → Terminal → Claude Code
After  (v0.0.5):    Agent Runtime Layer
                     ├ Agent Metadata (id, task, status)
                     ├ File Tracking (modified files, diff, jump)
                     ├ Activity Timeline (operations log)
                     └ Human Control v1 (pause/resume)
```

### 当前基线

- Workspace 多工作区 + Pane 分屏树
- 终端嵌入 (xterm.js + wasm ridge-term)
- Claude Code/OpenCode 集成 (tmux shim + HTTP teammate server)
- `PaneState` (`Idle` / `Busy` / `Starting`)
- `teammate_agent_pane_map: HashMap<agent_id → pane_id>`
- Agent 注册 (register_agent / release_pane HTTP routes + Tauri commands)
- 局部 prompt 历史 (localStorage per-pane)
- Claude Code 命令行历史文件 (history.jsonl)
- 文件 Diff 查看器 (DiffEditorModal)
- 文件系统 FsWatcher + Git watcher

**目标**：将当前的二元 `PaneState` (`idle`/`busy`) 升级为完整的结构化 Agent 元数据。

### 后端 — Rust

```rust
    Starting,
    Thinking,
    Editing,
    Running,
    Testing,
    Waiting,
    Failed,
```

## Task 2: Modified Files 追踪

**目标**：实时追踪每个 agent 修改了哪些文件，支持 diff / jump / rollback 预览。

### 实现策略

采用**混合追踪**方式：

1. **Git diff 快照**（主方案）— agent 启动时记录 git HEAD，轮询 `git diff --name-only` + `git diff --stat` 检测变化。优点是准确，缺点是有延迟
2. **文件系统 watcher 聚合**（辅助方案）— 现有 `FsWatcher` 捕获文件变更事件，关联到当前 busy agent。优点是实时，缺点是需要过滤噪声

优先级：Watcher 提供即时通知 → Git diff 提供精确状态

### 后端 — Rust

#### 新增模块 (`src-tauri/src/engine/agent_files.rs`)

```rust
pub struct AgentFileTracker {
    agent_id: String,
    pane_id: Uuid,
    workspace_id: Uuid,
    baseline_ts: SystemTime,       // agent 启动时间
    snapshots: Vec<FileSnapshot>,   // 最近 N 个状态快照
}

pub struct FileSnapshot {
    ts: SystemTime,
    files: Vec<FileChange>,
}

pub struct FileChange {
    path: PathBuf,
    status: 'modified' | 'created' | 'deleted',
    old_size: Option<u64>,
    new_size: Option<u64>,
}
```

#### 集成点

- `FsWatcher` 捕获 `fs-changed` → 查找当前 workspace 的 busy agent → 推入 `agent_files`
- 新增 Tauri 命令：`get_agent_files(pane_id) → Vec<FileChange>`
- 新增 HTTP route：`GET /api/v1/agent-files?pane_index=N`

### 前端 — Svelte

#### 新增组件 (`src/lib/components/AgentFileList.svelte`)

显示在 pane 标题栏下方（可折叠区域）：

```text

 ├ src/auth.ts          [M]  +120 -45   ▸ View Diff
 ├ src/login.tsx        [A]  +230 -0    ▸ Open File
 └ src/middleware.ts    [D]              ▸ Revert
```

每条文件项操作：
- `View Diff` → 打开 DiffEditorModal
- `Open File` → FileEditor 跳转
- `Revert` → git checkout 确认

#### Store (`src/lib/stores/agentFiles.ts`)

```typescript

  [paneId: string]: {
    files: FileChange[];
    lastUpdated: number;
  };
}
```

轮询：2s 间隔（可配置），agent busy 时才活跃。

## Task 3: Activity Timeline

**目标**：记录 agent 行为历史，让用户能回溯"AI 刚才做了什么"。

### 数据模型

```typescript
interface TimelineEvent {
  id: string;
  at: number;                           // timestamp
  type: 'agent_started' | 'agent_stopped' | 'read_file' | 'edited_file'
      | 'created_file' | 'deleted_file' | 'ran_command' | 'test_result'
      | 'agent_message' | 'status_change' | 'error';
  detail: string;                       // 人类可读描述
  file?: string;                        // 关联文件路径
  diff?: string;                        // 行级变化摘要
  prevStatus?: AgentStatus;             // 状态变更事件专用
  newStatus?: AgentStatus;
}
```

### 事件来源

1. **Agent 状态变化** — Task 1 的 metadata 变更自动产生事件
2. **文件变更** — Task 2 的文件追踪产生 `edited_file`/`created_file`/`deleted_file` 事件
3. **Agent 主动汇报** — `POST /api/v1/log-event` 让 agent 自行上报（需 agent 端配合）
4. **终端输出**（实验性）— 解析终端部分输出（如 test results）

### 存储

- 每 pane 存储，最多保留最近 500 条
- 内存存储（`Map<pane_id, TimelineEvent[]>`），可选 localStorage 持久化
- 工作区关闭时可选导出

### 前端

#### 新增 Store (`src/lib/stores/timeline.ts`)

```typescript
const timelineStore = writable<Record<string, TimelineEvent[]>>({});
```

侧边栏新面板，展示当前活跃 pane 的时间线：
- 时间线按时间倒序排列

- 筛选器：全部 / 仅文件变更 / 仅命令 / 仅错误

## Task 4: Human Control Layer v1

**目标**：Agent 不再是"发了就跑"；人类可以 pause / resume agent。

### Pause / Resume

#### 后端实现

- `POST /api/v1/pause-agent` → 向 pane 前台进程发 SIGSTOP（Unix）/ GenerateConsoleCtrlEvent（Windows）
- `POST /api/v1/resume-agent` → 发 SIGCONT（Unix）/ 无直接等价物（Windows 发送 Enter 恢复）
- Tauri 命令：`pause_agent(pane_id)`, `resume_agent(pane_id)`
- paused 状态下 `AgentStatus = Paused`，所有 file watch 继续

#### 前端实现

Pane 标题栏新增 pause/resume 按钮（仅在 agent 非 idle 时显示）：
- Running → 显示 ⏸（点击 pause）
- Paused → 显示 ▶（点击 resume）

Paused 状态时，Pane 内容区域显示半透明覆盖层 + 文字："Agent 已暂停 — 手动编辑后点击 Resume 继续"

#### Resume 上下文提示

Resume 时自主向 PTY 写入上下文感知提示：
```
[User modified files while you were paused:
  - src/auth.ts (modified)
  - src/types.ts (added)]
Continue where you left off.
```

### Manual Takeover

Pause 状态自动允许人类输入接管。用户可在终端直接操作：
1. 点击 Pause
2. 终端变为可输入状态
3. 用户执行任意命令或编辑文件
4. 点击 Resume → agent 收到变更摘要后继续
