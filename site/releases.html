v0.1.0
FIRST RELEASE 2026-04-30 在 GitHub 上查看 →

Ridge 的首个公开版本。在这一版里，分屏终端、代码编辑器、Git 提交图与智能体协作首次同时可用。

新增 · New

  • 递归分屏：水平、垂直、可嵌套，无层级限制；分屏可关闭、可拖拽改变大小。
  • 多工作区：可同时打开多个项目，每个工作区独立保留命令历史，切换不会中断后台进程。
  • 稳定的终端体验：支持 PowerShell / bash / zsh / cmd；Unicode、可点击超链接、长达数 MB 的可滚动命令历史。
  • 内嵌代码编辑器：可以把任意分屏切换为编辑器模式，文件、命令与 AI 输出在同一窗口内。
  • 文件浏览器：左侧栏文件树支持新建、重命名、删除、拖拽、键盘导航；右键菜单可在系统资源管理器中定位文件。
  • 跨工作区搜索：左侧栏搜索面板并行扫描所有打开的目录，支持大小写敏感、整词匹配、正则、glob 过滤、批量替换。
  • Git 提交图：直接渲染分支拓扑，可在分支选择器中切换或新建分支。
  • 分屏 Git 状态徽章：每个分屏标题栏显示当前分支、ahead / behind 数量与未提交改动数。
  • SCM 面板：浏览改动、暂存、提交、查看 diff。
  • Claude Code 智能体协作：从 Ridge 内的分屏启动智能体即可以多分屏模式工作；智能体可以列出、命名、新建、关闭分屏，并查询每个分屏当前的工作目录。
  • 主题与字体：内置三套配色与可选编辑器字体下拉。
  • 历史回放窗口：在每个分屏的标题栏可以打开历史窗口，浏览、搜索此前滚出屏幕的命令输出。

改进 · Improved

  • 仓库状态变化时自动刷新，无需手动重载或周期轮询。
  • 自动识别 git worktree 链接，子工作树的状态与主仓库一致。
  • 所有确认 / 输入对话框使用与 Ridge 自身风格一致的窗口，避免操作系统弹窗打断专注。
  • Windows 路径在前后端统一为正斜杠，资源管理器的目录列不会出现重复行。

已知限制 · Known

  • v0.1.0 只在 Windows 提供官方安装包；macOS / Linux 用户需要从源码构建。
  • 智能体协作目前以 Claude Code 为目标客户端，其它兼容多分屏会话协议的工具未做完整测试。
  • 站点上的演示截图与录屏正在补录中，部分仍是占位图。
