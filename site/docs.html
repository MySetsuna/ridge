文档

Ridge 是一个本地终端工作台。它把分屏终端、嵌入式编辑器、Git 可视化、智能体协作四样东西放进同一个原生应用里——四块田地，由一道道田埂隔开。

这份文档解释 Ridge 的核心概念、架构、以及你能怎样把它当作日常的开发台子用起来。

▎

这里只是入门。完整内部文档（PTY 实现、滚回设计、Agent 协议）见仓库 /docs。

安装

从 Releases 下载

访问 Releases 页面，下载与系统匹配的安装包：

  • Windows: ridge_0.1.0_x64-setup.exe 或 .msi
  • macOS / Linux: 请参考下面的源码构建（v0.1.0 暂未提供官方二进制）

从源码构建

需要 Node 18+, pnpm 9+, Rust 1.77+。Windows 下额外需要 MSVC + WebView2。

$ git clone git@github.com:MySetsuna/ridge.git
$ cd ridge
$ pnpm install
$ pnpm tauri dev   # 或 pnpm tauri build

第一次运行

启动后会看到一块默认的「田」——一个全屏的终端 Pane。试试这些动作：

  • Ctrl + \：水平切分（左右）
  • Ctrl + -：垂直切分（上下）
  • Ctrl + W：关闭当前 Pane
  • Ctrl + Shift + F：打开全工作区搜索

左侧栏顶部三个图标分别是：资源管理器（文件树）、源代码管理（SCM/Git）、搜索。

Pane / 田块

Pane 是 Ridge 最小的工作单元，对应你屏幕上的一个矩形区域。每个 Pane 有两种模式：

Mode说明
Terminal一个独立的 PTY 进程，运行你选择的 shell（pwsh / bash / zsh / cmd）
EditorMonaco 编辑器实例，带语言模型与 LSP 接口

每个 Pane 都有自己独立的 cwd、滚回缓冲区、Git 状态徽章。

分屏树（Pane Tree）

整个工作区用一棵递归树表示：

type PaneNode =
  | { type: 'leaf'; id: string }
  | { type: 'split';
      direction: 'horizontal' | 'vertical';
      children: PaneNode[];
      ratios: number[] };

每次切分相当于把一个叶子节点替换为含两个子叶子的 split 节点。深度无限。

工作区

工作区是一组共享同一个 Pane Tree 的 PTY 命名空间。你可以同时打开多个工作区——比如左边是后端项目，右边是文档 repo——每个工作区独立地保有自己的 PTY 进程列表、cwd 集合、SCM 状态。切换工作区不会杀死后台进程。

架构总览

Ridge = Tauri v2 (Rust) + SvelteKit + Svelte 5 (TypeScript) + Tailwind CSS v4。

  • 前端用 Svelte 5 runes ($state / $derived / $effect) 管理状态
  • 后端用 portable-pty 管 PTY、用 parking_lot::RwLock 共享 AppState
  • 前后端通过 Tauri IPC 通信（invoke() 命令 + listen() 事件）

前端

关键目录：

  • src/lib/components/：UI（SplitContainer、Pane、Explorer、GitGraph、SourceControl…）
  • src/lib/stores/：Svelte stores（paneTree.ts、paneCwdStore、paneGitStatus.ts…）
  • src/lib/plugins/：可注册到侧栏的插件 (registerSidebarPlugin)

后端

关键模块：

  • src-tauri/src/lib.rs：Tauri app 启动、事件总线、命令注册
  • src-tauri/src/state.rs：AppState（工作区、终端、滚回缓冲）
  • src-tauri/src/engine/pane_tree.rs：递归切分树
  • src-tauri/src/engine/pty.rs：PTY 句柄
  • src-tauri/src/commands/：所有 Tauri IPC 命令
  • src-tauri/src/bin/tmux.rs：内置 tmux 兼容 shim 二进制

IPC 模式

前端 → 后端用 invoke('command_name', payload) 走请求 / 响应。

后端 → 前端用事件订阅，关键的事件包括：

  • pty-output-{workspace_id}-{pane_id}：PTY 字节流
  • scm-repo-changed：仓库 .git/ 发生变化（去抖 500ms）
  • teammate-layout-changed：tmux shim 改动布局后通知

分块滚回（Block-paged Scrollback）

每个 Pane 后端维护 VecDeque<Arc<Vec<u8>>>，块大小 64 KiB，总上限 4 MiB。块带单调递增的 seq byte counter，方便确定性分页。

前端命令：

  • get_pane_scrollback_tail(pane_id, max_bytes) — 取最新 N 字节
  • get_pane_scrollback_before(pane_id, before_seq, max_bytes) — 加载更早

详见 docs/TERMINAL_SCROLLBACK.md。

侧栏搜索

第三个图标 (Ctrl+Shift+F) 打开搜索视图。它把 paneCwdStore 里所有不同的 cwd 并行喂给 text_search 命令，去重后展示。

  • 三个 toggle：大小写敏感 / 全词 / 正则
  • Glob 过滤：** / * / ? 翻译为正则在前端二次过滤
  • 支持 replace（按 root 分桶，调 replace_in_files）
  • 结果点击：fileEditorStore.openFile(path, { line, column })

Git / SCM

Git 状态由两条路径触发刷新：

  1. paneCwdStore 变化（280 ms 去抖 → discoverRepos）
  2. scm-repo-changed 事件（per-repo 去抖 → refreshStatus + loadGraph）

没有定时轮询、没有 workspace 切换强刷——完全靠文件监听。Linked worktree 的 .git 文件指向真正 git dir 的情形也被自动识别。

Agent 团队

Ridge 内置一个伪装成 tmux 的 shim 二进制（src-tauri/src/bin/tmux.rs → dist/teammate-shim/tmux.exe）。当你从 Ridge 内的 PTY 启动 Claude Code 时，Claude 会把 Ridge 当成 tmux 会话。

shim 把命令翻译成对本地 teammate HTTP API 的 POST/GET：

  • tmux list-panes → GET /api/v1/list-panes
  • tmux kill-pane → POST /api/v1/kill-pane
  • tmux rename-window → POST /api/v1/rename-pane
  • tmux display-message -p '#{...}' → 模板变量解析（pane_current_path, window_panes…）

详见 docs/AGENT_TEAMS_TEAMMATES.md。
